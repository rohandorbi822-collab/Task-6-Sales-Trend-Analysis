# Task 6: Sales Trend Analysis Using Aggregations

## Objective
Analyze monthly revenue and order volume using SQL aggregation functions.

## Tools Used
- PostgreSQL
- pgAdmin 4

## Dataset
- Orders.csv
- Details.csv

## SQL Concepts Used
- JOIN
- EXTRACT()
- SUM()
- COUNT(DISTINCT)
- GROUP BY
- ORDER BY
- LIMIT

## Queries Performed

### 1. Monthly Revenue and Order Volume
Calculated monthly revenue using SUM(amount) and order volume using COUNT(DISTINCT order_id).

### 2. Top 3 Months by Sales
Identified the top 3 months with highest sales revenue.

### 3. Specific Time Period Analysis
Analyzed revenue for a specific year (2018).

### 4. Total Revenue
Calculated overall revenue from the sales dataset.

## Results

- Highest Revenue Month: January 2018
- Total Revenue: 437771
- Successfully analyzed monthly sales trends and order volume.

## Outcome
Learned how to perform sales trend analysis using SQL aggregate functions and grouping techniques.
